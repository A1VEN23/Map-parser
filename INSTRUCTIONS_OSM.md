## 🔥 Как это работает

### Overpass API
- Используем бесплатный Overpass API от OpenStreetMap
- Запросы в формате: `amenity=dentist`, `amenity=cafe`, `amenity=car_repair`
- Поиск по границам городов или по названию

### Собираемые данные
- **Название** - из OSM тега `name`
- **Телефон** - из тегов `phone` или `contact:phone`
- **Сайт** - из тегов `website` или `contact:website`
- **Адрес** - из `addr:street`, `addr:housenumber`, `addr:city`

## 🎯 Умный фильтр "болей" (адаптирован для OSM)

Поскольку OSM не предоставляет рейтинги и отзывы, фильтр анализирует:

### 🔥 ГОРЯЧИЙ ЛИД, если:
1. **Нет телефона** - сложно связаться
2. **Нет сайта** - нет онлайн-присутствия
3. **Нет чат-бота** на сайте
4. **Нет формы записи** (для стоматологий/автосервисов)
5. **Нет контактной формы** на сайте
6. **Нет точного адреса**
7. **Нет названия** или слишком короткое

## ⚡ Производительность

- **Скорость:** ~2-3 секунды на город
- **Точность:** Высокая для крупных городов
- **Объем:** До 100+ результатов на категорию в большом городе

## 🚨 Важные замечания

1. **Качество данных** зависит от наполнения OpenStreetMap в вашем регионе
2. **Телефоны и сайты** не всегда указаны в OSM
3. **Адреса** могут быть неполными в небольших городах
4. **Overpass API** имеет ограничения на сложные запросы

## 🆘 Решение проблем

### Если ничего не найдено:
1. Проверьте название города (используйте официальное название)
2. Попробуйте английское название города
3. Увеличьте `max_results_per_category`

### Если медленно работает:
1. Уменьшите количество городов
2. Уменьшите `max_results_per_category`
3. Используйте более конкретные названия городов
